KDany's Steam Workshop Issue Tracker
=================================

Use this repository to report bugs and technical issues for my Steam Workshop maps.

This tracker is currently focused on **Counter-Strike 2** maps, and may expand to other games later.

Please note that this repository is dedicated to bug reports and technical feedback, and **new map requests are not accepted** here.

Language
--------

All issues and comments should be written in **English**.
If needed, please use a translator for your report text.

Code of Conduct
---------------

Please keep discussions respectful and useful:

- Do not insult, harass, or demean anyone.
- Do not intentionally submit duplicate issues.
- Do not use ALL CAPS in issue titles or descriptions.
- Do not repeatedly bump open issues to ask for updates.

Issue vs Bug
------------

On GitHub, an **issue** is the general report container.
A **bug** is one type of issue.

In this repository:

- Use **Bug Report** for things that are clearly broken or not working as intended.
- Use **Map Technical Feedback** for things that technically work, but still feel off, unclear, unbalanced, or have a confusing layout.

Typical examples:

- **Bug Report**: getting stuck, clipping, crashes, broken triggers, missing models/textures/sounds, script or entity errors.
- **Map Technical Feedback**: visibility problems, overly dark areas, confusing layout, weak sightlines, balance concerns, or spawn placement feedback.

How to Report a Bug
-------------------

Before opening a new issue:

- Search existing **issues** first, including closed ones: [Issue list](../../issues?q=is%3Aissue)
- If you find the same issue, add a thumbs up reaction to the original report instead of creating a duplicate.

When opening a new issue, include:

- A short, descriptive title
- The game and map name
- Clear reproduction steps
- Expected behavior and actual behavior
- Optional: Any useful screenshots or video if available

Submit an Issue
-----------

Use the **issue templates** to keep reports consistent, and the **GitHub Issues** tab to search existing reports or open a new one with the appropriate template.
